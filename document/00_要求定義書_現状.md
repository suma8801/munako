# 要求定義書（現状ソースから書き起こし）

## 1. システム概要

### 1.1 目的
同窓会の運営のため、出席確認・連絡方法の記録、同級生の検索を行うWebシステムである。

### 1.2 対象
- **宗高同窓会**（宗像高校 昭和56年卒業）の同窓生および幹事・管理者。

### 1.3 技術スタック
- **フレームワーク**: CakePHP 4.x
- **DB**: MySQL（Docker で初期データを投入）
- **認証**: CakePHP Authentication プラグイン（メールアドレス・パスワード）

---

## 2. ユーザ種別と権限

| 種別 | role_id | 説明 | 主な利用機能 |
|------|---------|------|----------------|
| 一般 | 1 | 同窓生（一般ユーザ） | ログイン後は検索ページ（homes/regacy）。メンバー検索、プロフィール表示・編集、パスワード変更。 |
| スタッフ | 2 | 同窓会幹事 | 一般の機能に加え、ダッシュボード、クラス別出欠一覧・編集。スタッフ・一般のユーザ登録（specialRegister）。 |
| 管理者 | 3 | システム管理者 | スタッフの機能に加え、全ロールのユーザ登録。初回のみ誰でもアクセス可能なユーザ登録（register）で管理者を作成可能。 |

### 2.1 認証・認可
- 全コントローラで認証を有効。未認証は原則としてログインへリダイレクト。
- 例外: `Homes/index` は未認証可（トップで一般/スタッフのログイン種別を選択するため）。
- スタッフ・管理者限定: `Homes/dashboard`, `Homes/classAttendance`, `Homes/editAttendance`, `Homes/updateAttendance`, `Users/specialRegister`。一般でアクセスした場合は regacy またはトップへリダイレクトしメッセージ表示。

## 3. 機能一覧（画面・アクション）

### 3.1 ホーム・入口（HomesController）

| アクション | URL例 | 認証 | 説明 |
|------------|--------|------|------|
| index | `/` | 不要 | トップ。一般ログイン/スタッフログインの選択。POSTでログイン種別に応じ Users/login へリダイレクト。 |
| regacy | `/homes/regacy` | 必要 | 一般ユーザ向けメイン画面。メンバー検索（クラス・名前/よみがな・物故者・出席者など）。 |
| dashboard | `/homes/dashboard` | 必要（スタッフ以上） | ダッシュボード。全体人数・生存者・物故者、開催年別参加者数グラフ（男性・女性・全体）、クラス別統計。クラスカードからクラス別出欠一覧へ遷移。 |
| classAttendance | `/homes/class-attendance/{class}` | 必要（スタッフ以上） | クラス別出欠一覧（表示のみ）。番号・名前・性別・出欠状況・メモ（20文字程度）・編集リンク。次回同窓会年（NEXT_YEAR）のデータのみ。 |
| editAttendance | `/homes/edit-attendance/{class}/{memberId}` | 必要（スタッフ以上） | 個別メンバーの出欠・メモ編集。出欠ステータス（セレクト）、メモ（textarea 4行）。保存で updateAttendance へPOST。 |
| updateAttendance | `/homes/update-attendance` | 必要（スタッフ以上） | POST で出欠状況・メモを更新。保存後は editAttendance にリダイレクト。 |

### 3.2 ユーザ（UsersController）

| アクション | URL例 | 認証 | 説明 |
|------------|--------|------|------|
| login | `/users/login` | 不要 | ログイン。成功時: スタッフ/管理者は dashboard、一般は regacy へ。 |
| logout | `/users/logout` | 要 | ログアウト。ログイン画面へリダイレクト。 |
| register | `/users/register` | 不要* | ユーザ登録。*管理者が未作成の初回のみ利用可（CSRF除外の条件あり）。初回は管理者(role_id=3)固定。 |
| specialRegister | `/users/special-register` | 要（スタッフ以上） | スタッフ・管理者によるユーザ登録。スタッフは一般・スタッフのみ選択可。管理者は全ロール選択可。 |
| forgotPassword | `/users/forgot-password` | 不要 | パスワード忘れ。メールアドレス入力でリセット用URL送信（ResetService）。 |
| resetPassword | `/users/reset-password/{token}` | 不要 | トークン付きURLで新パスワード設定。トークン・有効期限チェックあり。 |
| profile | `/users/profile` | 要 | プロフィール表示。 |
| edit | `/users/edit` | 要 | プロフィール編集。 |
| changePassword | `/users/change-password` | 要 | パスワード変更（現パスワード確認）。初回変更時は course_id を 0→1 に更新する処理あり。 |

### 3.3 その他
- **Pages**: `/pages/*` は CakePHP 標準の Pages::display。
- **エラー**: ErrorController / templates/Error で 400, 500 等を表示。

## 4. データモデル（主要テーブル）

### 4.1 users
- ログイン用。email（ログインID）, name, yomi, password（ハッシュ）, role_id, token, token_expire, created 等。
- role_id: 1=一般, 2=スタッフ, 3=管理者。
- テスト・コード上 course_id の参照あり（初回パスワード変更完了フラグ等）。

### 4.2 roles
- id, name（一般/スタッフ/管理者）, created.

### 4.3 members
- 同窓生マスタ。id, class, no, name, yomi, gone（物故者フラグ）, sex（性別: 1=男性, 0=女性）。

### 4.4 attend_statuses
- 出欠ステータスマスタ。id, name（出席・欠席・連絡先不明・連絡拒否・死亡）, created, modified.

### 4.5 reunion_attends
- 開催年ごとの出欠・メモ。id, year, member_id, attend_status_id, note, created, modified。
- 次回同窓会の年は定数 **NEXT_YEAR**（config/bootstrap.php で定義、例: 2027）で統一。

## 5. 画面遷移・URL まとめ

```
/                          → トップ（ログイン種別選択）
/users/login               → ログイン
/users/register            → 初回のみユーザ登録（管理者）
/users/forgot-password     → パスワード忘れ
/users/reset-password/:token → パスワードリセット

[一般ログイン後]
/homes/regacy              → メンバー検索

[スタッフ・管理者ログイン後]
/homes/dashboard           → ダッシュボード
/homes/class-attendance/:class        → クラス別出欠一覧
/homes/edit-attendance/:class/:memberId → 出欠・メモ編集
/homes/update-attendance   (POST)     → 出欠更新

[共通・要認証]
/users/profile             → プロフィール
/users/edit                → プロフィール編集
/users/change-password     → パスワード変更
/users/special-register    → スタッフ以上: ユーザ登録
/users/logout              → ログアウト
```

## 6. 非機能要件（現状実装ベース）

### 6.1 セキュリティ
- **CSRF**: CsrfProtectionMiddleware を適用。register は管理者未作成時の初回登録時のみスキップ条件あり。
- **パスワード**: ハッシュ化して保存。リセットはトークン＋有効期限で実施。

### 6.2 運用・設定
- **次回同窓会年**: `config/bootstrap.php` の定数 `NEXT_YEAR`（例: 2027）。出欠管理はこの年のみ対象。

### 6.3 サービス層
- **MemberSearchService**: メンバー検索（クラス・名前/よみ・物故者・出席者など）。
- **DashboardStatisticsService**: ダッシュボード用集計（総数・生存/物故・年別参加者・クラス別・2023年出席数等）。
- **ResetService**: パスワードリセットメール送信。
